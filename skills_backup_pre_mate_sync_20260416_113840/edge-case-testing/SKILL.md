# Skill: edge-case-testing

## Purpose
Execute edge and negative scenario validation for module workflows.

## Stage
`execution`

## Execution Rules
- Include invalid input, empty state, boundary, and conflict scenarios.
- Record expected vs actual behavior.
- Flag probable regression vs expected modernization changes.

## Dependencies
- `test-plan-generation`

## Outputs
- `edge-case-results.json`
- `result.json`
